## 4. Анализ производительности (Гипотетический)
Имея в наличии только результаты для одного размера матрицы (5x5), невозможно построить полноценный график зависимости времени от размера. Однако, используя данные из файла results.txt и понимание теоретической сложности алгоритма, можно сделать следующие предположения:

Теоретическая сложность: Классический алгоритм умножения матриц имеет сложность O(N³), где N — размерность матрицы. Количество операций (умножений и сложений) приблизительно равно 2N³.

Проверка на имеющихся данных:

Для N=5 теоретическое количество операций: 2 * 5³ = 2 * 125 = 250 операций.

Это значение полностью совпадает с числом операций, указанным в файле results.txt. Это служит дополнительным подтверждением того, что программа считает именно столько операций, сколько предполагает наивный алгоритм.

Для полноценного анализа временной зависимости, как в вашем примере README.md, необходимо провести серию экспериментов с матрицами больших размеров (например, 10x10, 50x50, 100x100, 200x200, 400x400) и зафиксировать время выполнения для каждого случая.
